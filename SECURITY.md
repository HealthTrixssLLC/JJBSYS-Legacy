# Security Policy

## Historical Archive Context

JJBSYS-Legacy is a historical archive and preservation project. Although the repository has been sanitized for public review, legacy source should still be treated as potentially sensitive during analysis.

## Reporting Guidance

Do not open public issues for suspected secrets, credentials, private hostnames, user IDs, operational procedures, or other security-sensitive findings.

Please report security concerns privately to the maintainers:

- Security contact: [jayallanbaker@gmail.com](mailto:jayallanbaker@gmail.com)

## Expectations for Reviewers

- Treat legacy source carefully and conservatively.
- Do not repost suspected sensitive values in public discussions.
- Assume any real-world adaptation requires independent security, compliance, compatibility, and operational review.

## Responsibility for Reuse

Any party adapting this code is responsible for independent security validation, compliance review, CICS and z/OS compatibility analysis, and local operational approval before any execution or deployment.
