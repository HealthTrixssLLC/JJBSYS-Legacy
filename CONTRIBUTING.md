# Contributing

JJBSYS-Legacy is a preservation and documentation project for a historical CICS/mainframe archive.

## What Contributions Are Most Helpful

Preferred contributions include:

- Documentation improvements.
- Component explanations and inventory refinement.
- Modernization mappings and review notes.
- Security and sanitization observations that can be handled safely.
- Historical context that helps explain how the source is organized and what problems it addressed.

Mainframe and CICS experts are especially encouraged to help classify source members and explain systems programming patterns.

## Boundaries for Contributions

- Do not submit real credentials, real hostnames, customer information, production JCL values, or private operational details.
- Source changes should be conservative.
- Do not rewrite original legacy members unless there is a clear documentation, correction, or sanitization reason.
- Avoid reframing the repository as production-ready software.

## Pull Request Expectations

- Explain what changed and why.
- Identify the files changed.
- Call out any public-readiness or sanitization impact.
- Confirm that no secrets, credentials, customer data, or private operational details were introduced.
